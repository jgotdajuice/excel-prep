  <how-to-verify>
Start the dev server (`npm run dev`) and verify each requirement:

**PROG-01 — Persistence:**
1. Go to /challenge, complete 2-3 challenges (at least one correct, one incorrect)
2. Go to /drill, complete a drill session
3. Hard refresh the browser (Cmd+Shift+R)
4. Go to /challenge — previously completed challenges should still show their status (correct/incorrect icons)
5. Go to /drill, start a new session — the session should be fresh (not mid-session), but allAnswers history should accumulate
6. Open DevTools → Application → Local Storage — verify `excelprep-challenge-v1` and `excelprep-drill-v1` keys exist with reasonable JSON

**PROG-02 — Per-function accuracy:**
7. Go to /progress
8. Verify overall stats show at the top (challenges completed count, drill answers count, overall accuracy %)
9. Verify per-function accuracy bars appear for functions you've attempted
10. Bars should be grouped by tier (Beginner, Intermediate, Advanced)
11. Colors: green for >= 80%, amber for >= 50%, red for < 50%
12. Functions you haven't attempted should NOT appear

**PROG-03 — Suggested next:**
13. If you have a function with < 80% accuracy and >= 2 attempts: verify a "Focus on this: [Function]" card appears
14. Click the suggestion — it should navigate to /challenge with the correct tier pre-selected
15. If all functions are above 80%: verify encouraging message appears instead

**LEARN-06 — Weighted drill queue:**
16. Deliberately get several questions wrong for one function (e.g., VLOOKUP)
17. Start a new drill session — VLOOKUP questions should appear noticeably more often than functions you've been getting right
18. Verify no duplicate questions appear in a single 10-question session

**Edge case — Private browsing:**
19. Open an incognito/private window
20. Navigate through the app (challenge, drill, progress) — no console errors should appear
21. Data will not persist (expected) but app should work normally
