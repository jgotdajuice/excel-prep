Modify `drillStore.ts` `startSession` action to use weighted random selection instead of pure random shuffle+slice:

1. **Import** `computeCategoryAccuracies` from `./progressSelectors` and `useChallengeStore` (to read challenges and statuses for accuracy computation).

2. **Replace the shuffle+slice in startSession** with a `buildWeightedQueue` function:

```
function buildWeightedQueue(
  candidates: Challenge[],
  accuracyMap: Map<string, number>,  // category → accuracy (0-1)
  targetCount: number,
): Challenge[] {
```

**Algorithm (from RESEARCH.md Pattern 3):**

a. Group candidates by category. Compute weight for each category: `weight = Math.max(0.05, 1 - accuracy)`. Categories with 0 attempts use accuracy = 0.5 (neutral weight = 0.5).

b. Guarantee minimum 1 question per category (if categories <= targetCount): pick one random challenge from each category.

c. Fill remaining slots (targetCount - guaranteed.length) via cumulative-weight random selection WITHOUT replacement (remove selected challenge from pool after each pick to avoid duplicates per Pitfall 4).

d. Shuffle the combined (guaranteed + extras) list and return slice(0, targetCount).

3. **In startSession:**
   - After filtering candidates by tier, compute accuracy map:
     ```
     const { challenges: allChallenges, statuses } = useChallengeStore.getState();
     const accuracies = computeCategoryAccuracies(allChallenges, statuses, get().allAnswers);
     const accuracyMap = new Map(accuracies.map(a => [a.category, a.accuracy]));
     ```
   - Call `buildWeightedQueue(filtered, accuracyMap, 10)` instead of `shuffled.slice(0, 10)`.
   - Convert to DrillQuestion via `map(challengeToDrillQuestion)` as before.

4. **Weighting math verification:** weight(30% acc) = 0.70, weight(80% acc) = 0.20. Ratio = 3.5x. This satisfies the CONTEXT.md spec of "roughly 2-3x."

5. **Edge cases:**
   - If candidates.length <= targetCount, return all candidates shuffled (no weighting needed).
   - If no accuracy data exists yet (first session ever), all categories get 0.5 default → equal weights → equivalent to current random behavior.
   - If user selected a specific tier, filtering already happened — weighting applies within that tier's functions only (per CONTEXT.md).

6. **Helper function `weightedPick`:** Cumulative-weight selection (see RESEARCH.md code example). Takes items and weights arrays, returns one item. Use this in a loop for the "extras" selection, removing picked item from pool each iteration.
