  <name>Task 1: Adapt SpreadsheetGrid for challenge mode</name>
  <files>src/components/SpreadsheetGrid.tsx</files>
  <action>
Modify `SpreadsheetGrid.tsx` to accept an optional `challenge` prop. When no challenge is provided, the grid behaves exactly as it does now (free-form practice mode). When a challenge is provided, the grid enters challenge mode.

**Props to add:**
```typescript
interface SpreadsheetGridProps {
  challenge?: Challenge;
  isLocked?: boolean;
  cellGrades?: Array<{ row: number; col: number; result: GradeResult }>;
  onGradeCell?: (row: number, col: number) => void;
}
```

**Challenge mode behavior:**

1. **Seed data loading:** When `challenge` prop changes, load `challenge.seedData` as the grid data. Use the `data` prop on HotTable driven by `challenge.seedData`. If this doesn't reliably sync with HyperFormula, use `useEffect` keyed to `challenge.id` to call `hot.loadData(challenge.seedData)` imperatively.

2. **Adaptive grid sizing:** Set `minRows`/`maxRows` to `challenge.seedData.length` and `minCols`/`maxCols` to `Math.max(...challenge.seedData.map(r => r.length))`. Remove the default 26 cols / 50 rows when in challenge mode.

3. **Seed cell locking via `cells` callback:** Build a `Set` of answer cell keys (`${row}:${col}`). In the `cells` callback:
   - If `isLocked` is true: return `{ readOnly: true }` for ALL cells
   - Else if key is in answerCellSet: return `{}` (editable)
   - Else: return `{ readOnly: true }` (seed cell)

4. **Answer cell border highlighting:** Register a custom renderer `answerCellRenderer` using `Handsontable.renderers.registerRenderer`. The renderer:
   - Calls `Handsontable.renderers.TextRenderer` first (base rendering)
   - Checks if `${row}:${col}` is in answerCellSet
   - If yes, applies outline based on grade status from `cellGrades` prop:
     - No grade yet: `td.style.outline = '2px solid #0066cc'` (blue — unattempted)
     - Correct: `td.style.outline = '2px solid #1a6b3c'` (green)
     - Incorrect: `td.style.outline = '2px solid #c0392b'` (red)
     - Error: `td.style.outline = '2px solid #c0392b'` (red, same as incorrect)
   - Apply `td.style.outlineOffset = '-2px'` to keep outline inside cell
   - In the `cells` callback, return `{ renderer: 'answerCell' }` for answer cells (merge with readOnly)

5. **Enter-only grading trigger:** Use the flag pattern from research:
   - `enterPressedRef = useRef(false)`
   - `beforeKeyDown` hook: if `e.key === 'Enter'` and active cell is an answer cell and grid is not locked → set `enterPressedRef.current = true`
   - `afterChange` hook: if `source === 'edit'` and `enterPressedRef.current === true` → for each changed answer cell, call `onGradeCell(row, col)`. Reset flag.
   - This ensures clicking away does NOT trigger grading (only Enter does).

6. **Preserve free-form mode:** When `challenge` is undefined, use the current behavior (26 cols, 50 rows, no cells callback, no custom renderer). Use early conditional branching in props to HotTable.

**Important anti-patterns to avoid:**
- Do NOT set `readOnly: true` on the HotTable root — use `cells` callback only
- Do NOT read cell value in `beforeChange` — it's stale
- Do NOT trigger grading on `source === 'loadData'`
- The `cells` callback must reference fresh `isLocked` state — if using closure, ensure HotTable re-renders when `isLocked` changes (pass `isLocked` as a key or use `updateSettings`)

**Component 1 — `src/components/RightPanel.tsx`:**

The right panel is the primary challenge UI. It shows:

1. **Challenge prompt:** `challenge.prompt` text, rendered as a paragraph. Include `challenge.title` as a heading.
2. **Hint button:** "Show Hint" button. When clicked, reveals `challenge.hintFunction` (e.g., "Try using: VLOOKUP"). Calls `showHint()` from store. Hidden once hint is shown (replaced by the hint text). Per locked decision: "no hint by default, Show hint button reveals the function name."
3. **Grading feedback:** Shown after grading. Three states per locked decisions:
   - Correct: green checkmark icon + "Correct!" text
   - Incorrect (wrong value): "Expected: $X — Got: $Y" with both values displayed. Format numbers to 2 decimal places.
   - Error: display nothing special here — the Excel error code is already visible in the grid cell per locked decision
   - For multi-cell challenges: show "Fill all answer cells before submitting" until all cells graded. Show per-cell results once all are graded.
4. **Explanation:** Expandable `<details>` element with summary "See explanation". Always starts collapsed (per locked decision). Shows `challenge.correctFormula` formatted as code, plus `challenge.explanation` text. Both visible whether user got it right or wrong.
5. **Navigation buttons:**
   - "Try Again" button (visible after grading, calls `retry()`)
   - "Next Challenge" button (visible after grading, calls `nextChallenge()`)
   - "Skip" button (always visible before grading, calls `skip()`)
6. **Timer:** Optional informational display. Show mm:ss format. No penalty, purely tracks time. Read `elapsedSeconds` from store. Update via 1-second `setInterval` calling `tick()`.
7. **Reset button:** Clears answer cell(s), calls `retry()`.

All state reads come from `useChallengeStore()`. All actions call store methods.
