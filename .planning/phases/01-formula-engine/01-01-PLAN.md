Use `toBeCloseTo` for floating-point comparisons (2 decimal places).

    Test cases (known Excel outputs — verify these are correct):

    1. **SUM**: `=SUM(A1:A3)` with cells [10, 20, 30] → 60
    2. **AVERAGE**: `=AVERAGE(A1:A3)` with cells [10, 20, 30] → 20
    3. **COUNT**: `=COUNT(A1:A4)` with cells [10, "text", 30, ""] → 2 or 3 (depending on empty cell handling — verify with Excel-compat config where evaluateNullToZero makes empty = 0, which COUNT should count as numeric → expect 3)
       NOTE: COUNT counts numeric values. With evaluateNullToZero=true, empty cell becomes 0 (numeric). "text" is non-numeric. So COUNT of [10, "text", 30, 0] = 3. Verify and adjust if needed.
    4. **COUNTIF**: `=COUNTIF(A1:A4,">15")` with cells [10, 20, 30, 5] → 2
    5. **VLOOKUP**: `=VLOOKUP(2,A1:B3,2,FALSE)` with data [[1,"Apple"],[2,"Banana"],[3,"Cherry"]] → "Banana"
    6. **INDEX/MATCH**: `=INDEX(B1:B3,MATCH(2,A1:A3,0))` with same data → "Banana"
    7. **SUMIFS**: `=SUMIFS(B1:B4,A1:A4,"East")` with data [["East",100],["West",200],["East",150],["North",50]] → 250
    8. **IF (simple)**: `=IF(A1>50,"Pass","Fail")` with A1=75 → "Pass"
    9. **IF (nested)**: `=IF(A1>=90,"A",IF(A1>=80,"B",IF(A1>=70,"C","F")))` with A1=85 → "B"
    10. **NPV**: `=NPV(0.1,B1:E1)+A1` with cells [rate=0.1, initial=-1000, cf1=300, cf2=400, cf3=400]. NPV(0.1, 300, 400, 400) + (-1000). Excel NPV(0.1, 300,400,400) = 300/1.1 + 400/1.21 + 400/1.331 ≈ 272.73 + 330.58 + 300.53 = 903.84. Plus -1000 = -96.16. Use `toBeCloseTo(-96.16, 0)` with 0 decimal tolerance to account for rounding. (Research doc says ~9.99 but that uses different cash flows — recalculate: NPV(10%, -1000, 300, 400, 400) where -1000 is the initial outlay passed as first cash flow to NPV: =NPV(A1,B1:E1) with B1=-1000. Actually follow the research doc pattern exactly: rate in A1=0.10, cash flows in B1=-1000, C1=300, D1=400, E1=400. Formula: =NPV(A1,C1:E1)+B1.)
    11. **IRR**: `=IRR(A1:A4)` with cells [-1000, 400, 400, 400] → ~9.7% (0.0970). Use `toBeCloseTo(0.0970, 2)`.
    12. **PMT**: `=PMT(A1,B1,C1)` with [rate=0.05/12, nper=60, pv=-10000] → ~188.71. Use `toBeCloseTo(188.71, 2)`.
    13. **XNPV**: `=XNPV(A1,B1:B4,C1:C4)` with rate=0.1, values=[-10000, 2750, 4250, 5250], dates=[2020-01-01, 2020-06-01, 2021-01-01, 2021-06-01]. XNPV should be ~1106. Use `toBeCloseTo` with 0 decimal tolerance. Dates must be entered as date serial numbers or date strings that HyperFormula can parse.

    Create `src/engine/formulaEngine.ts`:

    ```typescript
    import { HyperFormula } from 'hyperformula';

    export function buildExcelCompatEngine(): HyperFormula {
      return HyperFormula.buildEmpty({
        licenseKey: 'internal-use-in-handsontable',
        evaluateNullToZero: true,
        leapYear1900: true,
        nullDate: { year: 1899, month: 12, day: 31 },
        localeLang: 'en-US',
        functionArgSeparator: ',',
        decimalSeparator: '.',
        thousandSeparator: '',
        caseSensitive: false,
        useWildcards: true,
        useRegularExpressions: false,
        smartRounding: true,
      });
    }

    /** Finance functions supported in the grid (GRID-02 scope) */
    export const FINANCE_FUNCTIONS = [
      'SUM', 'AVERAGE', 'COUNT', 'COUNTIF',
      'VLOOKUP', 'INDEX', 'MATCH',
      'SUMIFS', 'IF', 'IFERROR',
      'NPV', 'IRR', 'PMT', 'XNPV',
    ] as const;

    export type FinanceFunction = typeof FINANCE_FUNCTIONS[number];
    ```
