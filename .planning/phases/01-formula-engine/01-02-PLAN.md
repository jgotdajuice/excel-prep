  <action>
    Build three components that together form the interactive spreadsheet.

    **SpreadsheetGrid.tsx:**
    - Import `HotTable` from `@handsontable/react-wrapper` (NOT `@handsontable/react` — that is the legacy package)
    - Import and call `registerAllModules()` from `handsontable/registry` (once, at module level)
    - Import Handsontable CSS: `handsontable/styles/handsontable.min.css` and `handsontable/styles/ht-theme-main.min.css`
    - Create the HyperFormula instance using `buildExcelCompatEngine()` from `src/engine/formulaEngine.ts`. Store it in a module-level variable or `useRef` — NOT in React state (React will try to proxy it and break it)
    - Configure HotTable with:
      - `formulas={{ engine: hfInstance, sheetName: 'Sheet1' }}`
      - `themeName="ht-theme-main"` for Excel-like styling
      - `colHeaders={true}` (A, B, C headers)
      - `rowHeaders={true}` (1, 2, 3 headers)
      - `width="100%"` and `height="100%"` — grid fills its container
      - `minCols={26}` (A through Z), `minRows={50}` — a generous default grid
      - `licenseKey="non-commercial-and-evaluation"`
      - `afterSelection` callback: reads `hfInstance.getCellFormula({ sheet: 0, row, col })` and `hfInstance.getCellValue({ sheet: 0, row, col })`, updates selected-cell state passed to FormulaBar
      - `afterBeginEditing` callback: sets up autocomplete trigger (see FunctionAutocomplete)
    - Do NOT wrap the grid in a `<form>` element (breaks Tab/Enter navigation — see Research pitfall 3)
    - Do NOT place focusable siblings adjacent without `tabindex="-1"`
    - Render FormulaBar above the grid, passing selected cell state as props
    - Render FunctionAutocomplete as an overlay (absolutely positioned) that shows when cell editor is active and input starts with `=`

    **FormulaBar.tsx:**
    - Accepts props: `{ formula: string | undefined, value: any, cellLabel: string }` where cellLabel is like "A1", "B3"
    - Displays a horizontal bar with:
      - Cell reference label on the left (e.g., "A1") in a small box
      - "fx" icon/label
      - Formula text (or computed value if no formula) in the main area
    - Styling: light gray background, border-bottom, monospace font for the formula text — resembling Excel's formula bar
    - The formula bar is read-only in Phase 1 (editing via formula bar is a nice-to-have for later). Display only.

    **FunctionAutocomplete.tsx:**
    - Import `FINANCE_FUNCTIONS` from `src/engine/formulaEngine.ts`
    - Props: `{ isVisible: boolean, filterText: string, position: { top: number, left: number }, onSelect: (funcName: string) => void }`
    - When `isVisible` is true and `filterText` is non-empty, filter `FINANCE_FUNCTIONS` by case-insensitive prefix match against `filterText`
    - Render a dropdown list positioned absolutely at the given `position`
    - Keyboard support: up/down arrows to select, Enter/Tab to confirm selection (call `onSelect` with the function name + opening parenthesis, e.g., "VLOOKUP(")
    - Escape hides the dropdown
    - Styling: white background, subtle shadow, border, each item highlighted on hover/keyboard-focus

    **Autocomplete trigger wiring (in SpreadsheetGrid):**
    - On `afterBeginEditing`, find the Handsontable editor's TEXTAREA element (it is the active element or can be found via `hot.getActiveEditor().TEXTAREA`)
    - Attach a `keyup` listener to this TEXTAREA
    - On each keyup, read the TEXTAREA value. If it matches `/^=([A-Z]*)$/i`, extract the prefix after `=` and show the autocomplete with that prefix as `filterText`
    - If the value does not match (user typed more, like `=SUM(`), hide the autocomplete
    - On autocomplete `onSelect`, replace the TEXTAREA value with `=` + selectedFunction (e.g., `=VLOOKUP(`), then refocus the editor
    - On `afterEditorClose` (or equivalent), hide autocomplete and remove the listener
    - Position the autocomplete dropdown below the active cell using `hot.getCell(row, col).getBoundingClientRect()`
  </action>
