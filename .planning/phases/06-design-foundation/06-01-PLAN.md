       ```css
       @theme {
         --color-brand: #1a6b3c;
         --color-brand-dark: #145530;

         --color-surface: #ffffff;

       }
       ```

    4. Update the existing `html, body, #root` rule to remove the old `font-family` declaration entirely — Tailwind v4 will apply `--font-sans` as the default font family via the `font-sans` utility and the base layer.

    5. Add HOT preflight repair CSS AFTER the `@theme` block but BEFORE the `.ht-theme-main` section. This prevents Tailwind's preflight from breaking HOT grid styles:
       ```css
       /* Tailwind preflight repair for Handsontable grid elements */
       .hot-container td,
       .hot-container th,
       .hot-container table {
         box-sizing: content-box;
         border-style: solid;
       }
       ```

    6. Do NOT modify any existing Handsontable CSS variable overrides (the `.ht-theme-main` block and everything below it). Do NOT modify any existing Challenge Page CSS classes. Only add the @theme block and HOT repair rules.

    IMPORTANT: Token names use `--color-brand` not `--color-brand-green` to keep utility classes clean (`bg-brand` not `bg-brand-green`). The `--color-brand-dark` maps to the existing `#145530` / `#1a3a2a` dark green used in headers. Use `#145530` for brand-dark (slightly lighter than the current `#1a3a2a` for better contrast on white).
