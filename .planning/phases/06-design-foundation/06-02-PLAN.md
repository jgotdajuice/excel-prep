    1. Create directory `src/components/ui/` if it does not exist.

    2. Create `src/components/ui/Button.tsx`:
       - Import `clsx` from 'clsx' (already in package.json)
       - Define `Variant` type: `'primary' | 'secondary' | 'ghost' | 'danger'`
       - Define `variantClasses` record mapping each variant to Tailwind classes consuming @theme tokens:
         - `primary`: `bg-brand text-white hover:brightness-110` (brand green button)
         - `secondary`: `bg-brand-light text-brand-dark border border-brand hover:bg-brand-light/80` (light green outline)
         - `ghost`: `bg-transparent text-muted border border-border hover:bg-base` (subtle outlined)
         - `danger`: `bg-red-600 text-white hover:bg-red-700` (destructive action)
       - Interface `ButtonProps` extends `React.ButtonHTMLAttributes<HTMLButtonElement>` with optional `variant` (default 'primary')
       - Export named `Button` component that renders a `<button>` with:
         - Base classes: `px-4 py-2 text-sm font-semibold rounded-btn transition-colors duration-150 cursor-pointer disabled:opacity-50 disabled:cursor-not-allowed`
         - Variant classes from the record
         - Caller's `className` merged last via `clsx`
         - All other props spread onto the button element
       - Use `clsx` (not `tailwind-merge`) since variants are non-conflicting by design.

    3. Create `src/components/ui/Card.tsx`:
       - Import `clsx` from 'clsx'
       - Interface `CardProps` extends `React.HTMLAttributes<HTMLDivElement>` with optional `padding` (default true) and optional `shadow` (default false)
       - Export named `Card` component that renders a `<div>` with:
         - Base classes: `bg-surface border border-border rounded-card`
         - Conditional: `p-6` when `padding` is true
         - Conditional: `shadow-sm` when `shadow` is true
         - Caller's `className` merged last via `clsx`
         - `children` and all other props spread onto the div
       - Keep it minimal — no hover states, no click handlers. It is a layout container.

    4. Do NOT create a StatCard component in this plan — that can be derived from Card in Phase 7 when ProgressPage is restyled. Keep the primitive count to the two specified in the requirements (VIS-04).

    5. Do NOT import or consume these components anywhere yet. They are created-but-unused, ready for Phase 7.
