Create a Zustand store for drill session state. Follow the exact same `create` pattern as challengeStore.

**State fields:**
```typescript
interface DrillStore {
  // Session config
  sessionTier: Tier | 'all';
  mode: 'typing' | 'mc';

  // Session questions
  questions: DrillQuestion[];
  currentQuestionIndex: number;

  // Timer
  secondsRemaining: number;
  timerActive: boolean;

  // Answers
  answers: DrillAnswerRecord[];

  // Session phase
  phase: 'idle' | 'active' | 'feedback' | 'review';
  feedbackStatus: 'correct' | 'incorrect' | null;

  // Actions
  startSession: (tier: Tier | 'all', mode: 'typing' | 'mc') => void;
  submitAnswer: (userAnswer: string) => void;
  tickTimer: () => void;
  advanceToNextQuestion: () => void;
  endSession: () => void;
  setMode: (mode: 'typing' | 'mc') => void;
}
```

Where `DrillAnswerRecord` is:
```typescript
type DrillAnswerStatus = 'correct' | 'incorrect' | 'timeout';
interface DrillAnswerRecord {
  questionIndex: number;
  userAnswer: string;
  status: DrillAnswerStatus;
}
```

**State machine (from research):**
- `idle` -> startSession() -> `active` (timer starts, first question shown)
- `active` -> submitAnswer() -> `feedback` (green/red flash, correct formula shown)
- `active` -> tickTimer() to 0 -> `feedback` (timeout, mark wrong, show correct)
- `feedback` (auto-advances after ~1.5s via useEffect in DrillPage)
- `feedback` -> advanceToNextQuestion() -> `active` (next question, timer reset) OR `review` (if last question)
- `review` -> endSession() -> `idle`

**startSession action:**
1. Import challenges from data and challengeToDrillQuestion from types
2. Filter challenges by `tier === selectedTier` (or all if 'all')
3. Shuffle filtered challenges: `[...filtered].sort(() => Math.random() - 0.5)`
4. Take first 10 (or all if fewer)
5. Map to DrillQuestion[] using `challengeToDrillQuestion`
6. Set `phase: 'active'`, `currentQuestionIndex: 0`, `answers: []`
7. Set `secondsRemaining` based on tier: beginner=45, intermediate=30, advanced=20. For 'all', use 30.

**submitAnswer action:**
1. Get current question from `questions[currentQuestionIndex]`
2. Normalize answer: for `formula` scope: `answer.trim().replace(/\s+/g, '').toUpperCase()`. For `function` scope: `answer.trim().toUpperCase()`.
3. Also strip leading `=` from both user answer and expected answer before comparing (accept both `=VLOOKUP(...)` and `VLOOKUP(...)`)
4. Compare normalized answers -> 'correct' or 'incorrect'
5. Push to `answers` array
6. Set `phase: 'feedback'`, `feedbackStatus: isCorrect ? 'correct' : 'incorrect'`

**tickTimer action:**
1. Decrement `secondsRemaining` by 1
2. If `secondsRemaining <= 0` AND `phase === 'active'`: auto-submit as timeout
   - Push `{ questionIndex: currentQuestionIndex, userAnswer: '', status: 'timeout' }` to answers
   - Set `phase: 'feedback'`, `feedbackStatus: 'incorrect'`

**advanceToNextQuestion action:**
1. If `currentQuestionIndex < questions.length - 1`: increment index, reset timer (based on question tier), set `phase: 'active'`, clear feedbackStatus
2. Else: set `phase: 'review'`

**endSession action:** Reset to idle state.

**setMode action:** Set `mode` field (can be changed before or during session).

Timer effect (same pattern as RightPanel.tsx):
```typescript
const intervalRef = useRef<ReturnType<typeof setInterval> | null>(null);
useEffect(() => {
  if (phase === 'active') {
    intervalRef.current = setInterval(() => tickTimer(), 1000);
  }
  return () => { if (intervalRef.current) clearInterval(intervalRef.current); };
}, [phase]);
```

Auto-advance effect:
```typescript
useEffect(() => {
  if (phase === 'feedback') {
    const timeout = setTimeout(() => advanceToNextQuestion(), 1500);
    return () => clearTimeout(timeout);
  }
}, [phase, currentQuestionIndex]); // include currentQuestionIndex to reset timeout on question change
```
