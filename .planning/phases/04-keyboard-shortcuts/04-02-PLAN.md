**3. `src/components/shortcuts/ShortcutDrill.tsx`** — Active drill (CRITICAL component):
- **Two drill directions** (per CONTEXT.md locked decision — bidirectional drills required):
  - **Action→Keys mode** (`drillDirection === 'action-to-keys'`): Shows action prompt, user presses correct key combination. Uses global keydown listener.
  - **Keys→Action mode** (`drillDirection === 'keys-to-action'`): Shows the key combination (formatted as "Ctrl + D"), user clicks one of 4 multiple-choice action options. No keydown listener needed — answer via click only. Generate 3 wrong-answer options from the same category (for plausibility). If fewer than 3 others in category, pull from adjacent categories. Shuffle the correct answer into a random position. On click, call `submitMultipleChoiceAnswer(selectedShortcutId)`.
- **Global keydown listener (Action→Keys only):** Use `useEffect` with native `document.addEventListener('keydown', handler, { capture: true })`. Only attach when `drillState === 'drilling'` AND `drillDirection === 'action-to-keys'`. Define `handleKeyDown` INSIDE the `useEffect` (per research Pitfall 2 — avoids stale closure and double-listener issues). Access store via `useShortcutStore.getState()` inside the handler, not via closure.
- **Key normalization:** Build pressed array from `e.ctrlKey` → 'Ctrl', `e.altKey` → 'Alt', `e.shiftKey` → 'Shift', `e.metaKey` → 'Cmd'. Normalize single-char `e.key` to uppercase. Skip modifier-only events (if `e.key` is Control/Alt/Shift/Meta, return early — do NOT submit).
- **preventDefault:** For interceptable shortcuts during drill: if `e.ctrlKey` and key is in `['s','p','d','r','b','i','u','f','z','a','l','1','9']` → `e.preventDefault()`. Also preventDefault for Escape, F2, F4, F5, F9, Shift+F3.
- **Escape handling (both directions):** If `e.key === 'Escape'`, `e.preventDefault()`, call `requestQuit()` on store, return (do NOT submit as answer). In Keys→Action mode, still attach a minimal keydown listener ONLY for Escape (no key grading).
- **Quit confirmation:** When `showQuitConfirm` is true, render an inline confirmation panel: "Quit this session? [Yes, quit] [Keep going]". Yes calls `confirmQuit()`, No calls `cancelQuit()`. While quit dialog is shown, the keydown listener should still be active but should only respond to Escape (toggle confirm off) — do NOT grade keypresses while dialog is showing.
- **Progress indicator:** "Question {currentIndex + 1} of {queue.length}" at top
- **Prompt display:**
  - Action→Keys: Show `queue[currentIndex].action` as the main prompt text (large, centered). Below it show `queue[currentIndex].financeContext` in smaller muted text.
  - Keys→Action: Show the key combination (e.g., "Ctrl + D") as the main prompt. Below show "What does this shortcut do?" and render 4 clickable option buttons with action names.
- **Timer display (timed mode):** Show `timeRemaining` seconds as a countdown. Use `useEffect` with `setInterval` ticking every 1000ms, calling `useShortcutStore.getState().tick()`. Only run interval when `drillMode === 'timed'` AND `drillState === 'drilling'`. Clear interval on cleanup.
- **Capture mode indicator:** Show a small badge/banner "Capturing shortcuts" when drill is active AND direction is action-to-keys (per CONTEXT.md locked decision). Not needed for multiple-choice direction.
- **Feedback display:** When `drillState === 'feedback'`, render `<ShortcutFeedback />` overlay/inline. After feedback delay (1s correct, 2s incorrect), call `advanceToNext()` via `setTimeout` in a `useEffect` that triggers when `drillState === 'feedback'`.

**4. `src/components/shortcuts/ShortcutFeedback.tsx`** — Feedback display:

**5. `src/components/shortcuts/ShortcutSummary.tsx`** — End-of-session summary:
