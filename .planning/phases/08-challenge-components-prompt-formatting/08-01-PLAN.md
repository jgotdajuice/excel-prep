  <name>Task 1: Create formatPrompt utility for structured prompt rendering</name>
  <files>src/utils/formatPrompt.ts</files>
  <action>
Create `src/utils/formatPrompt.ts` that exports:

1. **`PromptSection` type** — a discriminated union:
   ```ts
   type PromptSection =
     | { type: 'scenario'; text: string }
     | { type: 'data'; text: string }
     | { type: 'task'; text: string }
   ```

2. **`formatPrompt(raw: string): PromptSection[]`** — parses a raw prompt string into structured sections:
   - **Scenario**: Everything before the first data block (table or bullet list). Trim leading/trailing whitespace.
   - **Data block**: Detected by either:
     - Lines starting with `|` (markdown table rows) — collect all consecutive `|`-prefixed lines as one data block
     - Lines starting with bullet chars (`•` or `- `) that contain data patterns (numbers, $ signs, slashes separating fields) — collect consecutive bullet lines as one data block
   - **Task**: Everything after the data block. This is the instruction telling the user what formula to write. Often starts with "Enter a...", "Write a...", "Calculate the...", etc.

   **Parsing algorithm:**
   - Split the raw string on `\n`
   - Walk lines, classify each as scenario/data/task based on position relative to data block
   - Lines before first data line = scenario
   - Consecutive data lines (|... or bullet with data) = data block
   - Lines after data block ends = task
   - If no data block detected (edge case), return entire string as a single scenario section

   **Edge cases to handle:**
   - Some prompts have the task instruction on the same line as the last data row — split at the blank line after data
   - Blank lines between sections should be trimmed
   - The data block for bullets uses `•` (unicode bullet, not ASCII)

3. **`PromptDisplay` React component** — renders `PromptSection[]` as structured JSX:
   ```tsx
   export function PromptDisplay({ sections }: { sections: PromptSection[] }) {
     return (
       <div className="flex flex-col gap-3">
         {sections.map((s, i) => {
           if (s.type === 'scenario')
             return <p key={i} className="text-[13px] text-text-primary/80 leading-relaxed">{s.text}</p>;
           if (s.type === 'data')
             return <pre key={i} className="text-[12px] font-mono bg-base border border-border rounded-card px-3 py-2 overflow-x-auto whitespace-pre text-text-primary/90 leading-snug">{s.text}</pre>;
           if (s.type === 'task')
             return <p key={i} className="text-[13px] font-medium text-brand-dark leading-relaxed">{s.text}</p>;
           return null;
         })}
       </div>
     );
   }
   ```

   The scenario text is normal body copy. The data block renders as a monospace code block with bg-base background and border. The task instruction is slightly emphasized with brand-dark color and font-medium weight.

Test against these prompt patterns from the challenge data:
- `sum-01`: Has a markdown table (`| Row | A...`) then a task line
- `if-01`: Has bullet points (`•`) with data then a task line
- `iferror-01`: Has bullet points with employee data
- `sumifs-01`: Has a markdown table

All should parse into 3 sections (scenario, data, task).
